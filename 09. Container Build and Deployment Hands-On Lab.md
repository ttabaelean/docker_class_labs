# 1. [종합실습 1] 리눅스 서버 기반 2-Tier 서비스 운영 (Non-Container)

💡**실습 시나리오**

- Server 1과 Server 2를 스냅샷 기능을 이용해 도커 컨테이너 설치 초기 상태로 되돌립니다.
- 컨테이너를 전혀 사용하지 않고, Server 1에는 Java Application(Spring Petclinic)을, Server 2에는 Database(MariaDB)를 각각 독립적으로 직접 설치하여 연동 운영합니다.

## 📌 실습 전 핵심 개념 학습 및 용어 정리

인프라 엔지니어가 가상 서버 위에 서비스를 올리기 위해 반드시 알아야 할 개발 시스템 핵심 용어입니다.

- **Java Spring Boot (스프링 부트)**
    - 현대 엔터프라이즈 웹 애플리케이션을 만드는 가장 대표적인 오픈소스 소스코드 프레임워크(뼈대)입니다.
    - 가장 큰 특징은 애플리케이션 내부에 웹 서버(Tomcat)를 기본 탑재하고 있어, 별도의 외산 WAS(제우스, 와일드플라이 등)를 설치하지 않고도 자바 명령어만으로 웹 서비스를 구동할 수 있습니다.
- **Git**
    - 전 세계 개발자들이 소스코드를 저장하고 공유하는 원격 저장소(GitHub)에서 소스코드 폴더 전체를 리눅스 서버로 안전하게 다운로드해 주는 파일 수집 및 형상 관리 도구입니다.
- **Maven Wrapper를 통한 패키징 빌드 (./mvnw)**
    - 개발자가 텍스트 형태로 짠 복잡한 자바 소스코드 파일들과 수많은 외부 라이브러리들을 하나로 모으고 컴파일하여, 리눅스 운영체제 위에서 단독 실행이 가능한 하나의 압축 파일(`.jar`)로 조립해 주는 "인프라 빌드 자동화 로봇"입니다

### 1.1 [Server 2] MariaDB(MySQL) 데이터베이스 설치 및 구성

Server 2(IP: 192.168.100.20)에 접속하여 백엔드 데이터베이스 인프라를 구축합니다.

- **STEP1: MariaDB 패키지 설치 및 데몬 가동**
    
    ```bash
    sudo dnf install mariadb mariadb-server -y
    sudo systemctl enable --now mariadb.service
    ```
    
- STEP2: 데이터베이스 root 초기 패스워드 설정
    
    ```bash
    mysql -u root
    ```
    
    - [MariaDB 프롬프트 진입 후 아래 쿼리를 실행합니다]
    
    ```sql
    ALTER USER 'root'@'localhost' IDENTIFIED BY 'pass123!@#';
    FLUSH PRIVILEGES;
    EXIT;
    ```
    
- STEP 3: 원격 연동을 위한 전용 DB 및 petclinic 유저 생성
    - 외부 WAS(Server 1)가 원격 접속할 수 있도록 Host 권한을 모든 대역('%')으로 개방하는 것이 핵심입니다.
    
    ```bash
    mysql -u root -p'pass123!@#'
    ```
    
    - [MariaDB 프롬프트 진입 후 아래 쿼리를 순서대로 실행합니다]
    
    ```bash
    CREATE DATABASE petclinic;
    CREATE USER 'petclinic'@'%' IDENTIFIED BY 'petclinic1234';
    GRANT ALL PRIVILEGES ON petclinic.* TO 'petclinic'@'%';
    FLUSH PRIVILEGES;
    EXIT;
    ```
    
- STEP 4: Server 2 OS 방화벽 포트 개방
    - 외부에서 MariaDB 기본 포트(3306)로 접근할 수 있도록 방화벽 규칙을 허용합니다. 만약, 시스템의 방화벽이 비활성화 되어있다면 이 실습 skip
    
    ```bash
    sudo firewall-cmd --state
    sudo firewall-cmd --permanent --add-port=3306/tcp
    sudo firewall-cmd --reload
    ```
    

### 1.2 [Server 1] Java 17 런타임 설치 및 애플리케이션 빌드/운영

Server 1(IP: 192.168.100.10)에 접속하여 환경을 구성하고 소스코드를 빌드합니다.

- STEP1: OpenJDK 17 개발 도구 패키지 설치
    
    ```bash
    sudo dnf install java-17-openjdk java-17-openjdk-devel -y
    java --version
    ```
    
- STEP2: Git 설치 및 Spring Petclinic 오픈소스 프로젝트 복제
    
    ```bash
    sudo dnf install git -y
    git clone https://github.com/spring-projects/spring-petclinic.git
    cd spring-petclinic
    ```
    
- STEP3: 원격 데이터베이스 외부 연결 프로퍼티 수정
    - 애플리케이션이 로컬이 아닌 Server 2(192.168.100.20)의 DB 서버를 바라보도록 연동 주소를 세팅합니다.
    
    ```bash
    vi src/main/resources/application-mysql.properties
    ```
    
    - [파일 내부 내용을 아래 명세대로 정확히 수정 후 저장합니다]
    
    ```bash
    database=mysql
    spring.datasource.url=jdbc:mysql://192.168.100.20:3306/petclinic
    spring.datasource.username=petclinic
    spring.datasource.password=petclinic1234
    spring.sql.init.mode=always
    ```
    
- STEP 4: Maven Wrapper를 통한 패키징 빌드 및 실행
    - 단독 실행이 가능한 내장 톰캣 JAR 파일로 컴파일한 뒤, mysql 프로필을 활성화하여 가동합니다.
    
    ```bash
    ./mvnw package -DskipTests
    cd target
    java -jar -Dspring.profiles.active=mysql spring-petclinic-*.jar
    ```
    
- STEP 5: Server 2 OS 방화벽 포트 개방
    
    ```bash
    sudo firewall-cmd --state
    sudo firewall-cmd --permanent --add-port=8080/tcp
    sudo firewall-cmd --reload
    ```

### 2.3 도커 빌드 명세서 (Dockerfile) 준비

- 도커 빌드 명세서 (Dockerfile) 준비
    - 소스코드 수집 효율을 위해 불필요한 빌드 파일은 스킵하도록 차단 파일(.dockerignore)과 컨테이너 설계도(Dockerfile)를 작성합니다.
    
    ```docker
    cat << 'EOF' > .dockerignore
    target
    .git
    EOF
    ```
    
    ```docker
    cat << 'EOF' > Dockerfile
    FROM eclipse-temurin:17-jdk-jammy
    WORKDIR /app
    
    # Maven Wrapper 라이브러리 레이어 사전 복사
    COPY .mvn/ .mvn
    COPY mvnw pom.xml ./
    
    # 의존성 내부 라이브러리 선행 캐싱 다운로드
    RUN ./mvnw dependency:resolve
    
    # application-mysql.properties 설정 파일이 포함된 src 전체를 컨테이너로 전달
    COPY src ./src
    
    # 소스코드 복사 및 최종 패키징 컴파일 빌드
    RUN ./mvnw package -DskipTests
    
    # 컨테이너 외부 개방 포트 정의 및 구동 프로세스 지정 (mysql 프로필 활성화 구동)
    EXPOSE 8080
    CMD ["java", "-Dspring.profiles.active=mysql", "-jar", "target/spring-petclinic-4.0.0-SNAPSHOT.jar"]
    EOF
    ```

### 3.1 멀티 스테이지 빌드 기반의 경량화 Dockerfile 재구성

- 이전에 구성한 수동 환경 및 폴더 일괄 청소
    
    ```bash
    # 1. 실행 중인 모든 도커 컨테이너 강제 종료 및 삭제
    docker rm -f $(docker ps -aq)
    
    # 2. 기존 베어메탈 실습 소스 폴더 완전 삭제
    cd ~
    rm -rf ~/spring-petclinic
    
    # 3. GitHub 원격 저장소로부터 순정 상태의 소스코드 재다운로드
    git clone https://github.com/spring-projects/spring-petclinic.git
    
    # 4. 새로 다운로드한 워크스페이스 디렉토리로 이동
    cd ~/spring-petclinic
    ```
    
- 멀티 스테이지 컨테이너 빌드
    - 빌드 스테이지(build)와 운영 스테이지(production)를 분리하여 컨테이너 용량을 최소화합니다.
    
    ```docker
    cat << 'EOF' > Dockerfile
    # Stage 1: Build & Packaging (빌드 스테이지)
    FROM eclipse-temurin:17-jdk-jammy AS build
    WORKDIR /app
    COPY .mvn/ .mvn
    COPY mvnw pom.xml ./
    RUN ./mvnw dependency:resolve
    COPY src ./src
    RUN ./mvnw package -DskipTests
    
    # Stage 2: Production Runtime Minimal Image (경량화 실행 스테이지)
    FROM eclipse-temurin:17-jre-jammy AS production
    WORKDIR /app
    EXPOSE 8080
    COPY --from=build /app/target/spring-petclinic-*.jar /app/spring-petclinic.jar
    CMD ["java", "-Djava.security.egd=file:/dev/./urandom", "-Dspring.profiles.active=mysql", "-jar", "/app/spring-petclinic.jar"]
    EOF
    ```
    

### 3.2 컴포즈 전용 내부 연동용 로컬 프로퍼티 파일 별도 작성

- 단일 서버 환경 내에서 가동될 컨테이너 DB(`petclinic-db`) 주소를 가리키는 외부 설정 파일을 호스트 디렉토리에 미리 준비해 둡니다.
    
    ```bash
    cat << 'EOF' > src/main/resources/application-mysql.properties
    database=mysql
    spring.datasource.url=jdbc:mysql://petclinic-db:3306/petclinic
    spring.datasource.username=petclinic
    spring.datasource.password=petclinic
    spring.sql.init.mode=always
    EOF
    ```
    

### 3.3 올인원 인프라 아키텍처 토폴로지 설계 (docker-compose.yml)

- 다음의 docker-compose.yml 파일을 생성합니다.
    
    ```sql
    cat << 'EOF' > docker-compose.yml
    version: '3.8'
    
    services:
      petclinic-was:
        build:
          context: .
        container_name: petclinic-was
        ports:
          - "8080:8080"
        volumes:
          # 📌 호스트의 생성 파일명과 좌측 경로를 완벽하게 일치시켜 컨테이너 내부로 주입합니다.
          - ./src/main/resources/application-mysql.properties:/app/config/application-mysql.properties
        command: [
          "java", 
          "-Djava.security.egd=file:/dev/./urandom", 
          "-Dspring.profiles.active=mysql", 
          "-jar", 
          "/app/spring-petclinic.jar", 
          "--spring.config.additional-location=file:/app/config/"
        ]
        depends_on:
          - petclinic-db
        networks:
          - infra-net
    
      petclinic-db:
        image: mysql:8.0
        container_name: petclinic-db
        ports:
          - "3306:3306"
        environment:
          - MYSQL_ROOT_PASSWORD=root
          - MYSQL_ALLOW_EMPTY_PASSWORD=true
          - MYSQL_USER=petclinic
          - MYSQL_PASSWORD=petclinic
          - MYSQL_DATABASE=petclinic
        volumes:
          - petclinic_mysql_data:/var/lib/mysql
        networks:
          - infra-net
    
    volumes:
      petclinic_mysql_data:
    
    networks:
      infra-net:
        driver: bridge
    EOF
    ```
